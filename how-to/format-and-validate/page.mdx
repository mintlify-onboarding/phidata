## Format

Formatting the codebase using a set standard saves us time and mental energy. Phidata templates are pre-configured with <LinkOpenNewTab href="https://docs.astral.sh/ruff/" text="ruff" /> that you can run using a helper script or directly.

<CodeGroup>

```bash Terminal
./scripts/format.sh
```

```bash Ruff
ruff format .
```

</CodeGroup>

## Validate

Linting and Type Checking add an extra layer of protection to the codebase. We highly recommending running the validate script before pushing any changes.

Phidata templates are pre-configured with <LinkOpenNewTab href="https://docs.astral.sh/ruff/" text="ruff" /> and <LinkOpenNewTab href="https://mypy.readthedocs.io/en/stable/" text="mypy" /> that you can run using a helper script or directly. Checkout the `pyproject.toml` file for the configuration.

<CodeGroup>

```bash Terminal
./scripts/validate.sh
```

```bash Ruff
ruff check .
```

```bash Mypy
mypy .
```

</CodeGroup>
